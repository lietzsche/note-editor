# DESIGN GUIDELINES

- 문서 버전: v1.0
- 작성일: 2026-04-11
- 상태: Draft

## 1. 목적

노트 편집 서비스 UI/UX의 일관성과 확장성을 유지하기 위한 디자인 기준을 정의한다.

## 2. 디자인 원칙

1. 명확성: 화면 목적과 주요 행동이 즉시 이해되어야 한다.
2. 안정성: 노트 전환/저장/삭제 같은 고위험 행동은 상태를 명확히 보여줘야 한다.
3. 연속성: 주요 흐름(목록 -> 편집 -> 목록 복귀)의 인지 부하를 최소화한다.
4. 일관성: 동일한 의미의 상태/버튼/문구는 화면 전반에서 동일하게 사용한다.

## 3. 정보 구조(IA)

1. 인증 영역
- 로그인
- 회원가입

2. 노트 영역
- 그룹(폴더/카테고리) 목록
- 노트 목록
- 노트 편집 뷰
- 그룹 관리 액션(생성/이름 변경/삭제)
- 편집 보조 정보(실시간 글자 수)
- 편집 유틸 액션(전체 복사)

3. 공통 영역
- 사용자 메뉴(로그아웃/세션 상태)
- 오류/성공 피드백

## 4. 핵심 UX 규칙

1. 노트 독립성
- 노트 A/B 전환 시 내용이 섞이지 않아야 한다.
- 저장 완료 전/후 상태를 시각적으로 구분한다.

2. 그룹 기반 탐색
- 그룹 선택 시 해당 그룹 노트만 표시한다.
- 그룹 미지정 노트도 접근 가능해야 한다.
- 그룹 생성/이름 변경/삭제가 가능해야 한다.
- 그룹 삭제 시 소속 노트는 기본 그룹(미분류)로 이동한다는 정책을 고정하고 명확히 안내해야 한다.
- 기본 그룹 자체는 삭제 불가 정책을 유지해야 한다.

3. 파괴적 액션
- 삭제는 명시적 사용자 액션으로만 실행한다.
- 삭제 전 확인 단계 또는 Undo 전략을 제공한다.

4. 전환/저장 충돌 처리
- 미저장 변경이 있을 때 노트 전환 시 `저장 후 이동 / 버리고 이동 / 취소` 선택지를 제공한다.
- 저장 실패 상태에서는 실패 원인과 재시도 액션을 노출한다.
- 다중 탭/세션 충돌이 감지되면 마지막 저장 시각과 함께 `덮어쓰기/취소` 중 하나를 선택하게 한다.
- `취소` 선택 시 현재 편집값은 유지되어야 한다.

5. 편집 생산성
- 글자 수는 편집기 근처에서 실시간으로 확인 가능해야 한다.
- 전체 복사 버튼은 찾기 쉬운 위치(편집 툴바/헤더)에 배치한다.
- 복사 성공/실패 피드백은 짧고 명확하게 노출한다.

## 5. 컴포넌트 기준(초안)

1. 필수 컴포넌트
- GroupList
- NoteList
- NoteEditor
- SaveStatusIndicator
- ConfirmDeleteDialog
- CharacterCountIndicator
- CopyAllButton

2. 상태 모델
- loading
- empty
- error
- ready

## 6. 접근성 기준(최소)

1. 키보드만으로 주요 흐름(그룹 선택/노트 이동/편집/저장) 수행 가능
2. 포커스 순서가 시각적 순서와 일치
3. 버튼/입력/오류 메시지는 스크린리더로 식별 가능

## 7. 반응형 기준

1. Desktop: 그룹/노트 목록 + 편집기 2~3패널 레이아웃 허용
2. Mobile: 단일 패널 전환형 레이아웃
3. 작은 화면에서 편집 행동(저장/삭제) 버튼 접근 가능성 보장
