## Summary

Implemented drag-and-drop reorder for:

1. Sidebar group order
2. Note list order in `All notes`
3. Note list order in group-filtered views

This change also added the missing group reorder persistence API and replaced the previous button-based note reorder flow with DnD-only interaction.

## Key Changes

1. Added `POST /api/groups/reorder` and client support in `api.groups.reorder`
2. Added sortable DnD components for groups and notes with explicit drag handles
3. Kept note reorder persistence on the existing `POST /api/notes/reorder` contract
4. Preserved group-scoped note reorder cache merge into `All notes`
5. Split reorder state into separate group and note status paths
6. Added integration coverage for group reorder success and validation failures
