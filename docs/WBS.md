# WBS: note-editor 개발 작업 분해 구조

- 문서 버전: v1.0
- 작성일: 2026-04-11
- 기준 문서: `docs/product/PRD.md`, `docs/testing/TEST_PLAN.md`, `docs/specs/*`, `docs/operations/DEPLOY.md`

## 1. 목적

SR-1 ~ SR-11을 구현 가능한 단위로 분해하고, 선행관계/산출물/완료기준을 명확히 해 개발 착수와 진행 관리를 표준화한다.

## 2. 범위

1. 인프라 부트스트랩(Cloudflare Pages + D1 + Migration Framework)
2. 인증/노트/그룹/전환 독립성/편집 생산성 기능 구현
3. 테스트 자동화와 배포 검증 체계 구축

## 3. 단계별 WBS

| WBS ID | 작업 | 상세 작업 | 산출물 | 선행 | 완료 기준 |
|---|---|---|---|---|---|
| 0 | 착수 준비 | 브랜치 전략, 코딩 규칙, 이슈 템플릿, 기본 스크립트 확인 | 개발 가이드/초기 보드 | 없음 | 팀 공통 작업 방식 합의 완료 |
| 1 | 인프라 부트스트랩 | Pages 프로젝트, D1 생성/바인딩, `wrangler.toml`, `migrations/` 초기화 | 실행 가능한 로컬/원격 인프라 | 0 | `npm run dev`, D1 연결, 마이그레이션 실행 성공 |
| 2 | 인증/세션(SR-1, SR-7) | signup/login/logout/me, 세션 쿠키 보안, 로그인 속도 제한, 인증 로그 | Auth API + 미들웨어 | 1 | TS-01/TS-05/TS-06 통과 |
| 3 | 노트 코어(SR-2~SR-6) | 목록/생성/수정/삭제/정렬, 입력 검증, 정렬 payload 적용 | Notes API + 저장 로직 | 2 | TS-02/TS-03 통과 |
| 4 | 그룹 관리(SR-8) | 기본 그룹 자동 생성, 그룹 CRUD, 그룹 필터/이동, 기본 그룹 삭제 제한 | Groups API + 그룹 연동 UI/API | 3 | TS-07 통과 |
| 5 | 전환 독립성/충돌(SR-9) | 에디터 상태관리(`dirty/saving/conflict`), 전환 가드, 충돌 `덮어쓰기/취소` 처리 | 전환/충돌 처리 모듈 | 3,4 | TS-08/TS-09 통과 |
| 6 | 편집 생산성(SR-10, SR-11) | 실시간 글자 수(grapheme cluster), 전체 복사, 성공/실패 피드백 | 생산성 UI/유틸 | 5 | TS-10/TS-11 통과 |
| 7 | 디자인 시스템 적용 | 토큰 적용, 컴포넌트 규칙 준수, 접근성/반응형 체크 | 일관 UI 컴포넌트 | 3,4,5,6 | 디자인 체크리스트 충족 |
| 8 | 테스트 자동화 | 단위/통합/E2E 우선 시나리오 자동화, CI 게이트 구성 | 테스트 파이프라인 | 2~7 | PR 단계 테스트 게이트 통과 |
| 9 | 배포/운영 검증 | 원격 마이그레이션, 빌드/배포, 스모크/확장 회귀, 보안/로그 점검 | 배포 기록 + 검증 결과 | 8 | DEPLOY 기준 점검 완료 및 승인 |

## 6. 병행 가능 작업

1. WBS 7(디자인 시스템 적용)은 WBS 3~6과 병행 가능
2. WBS 8(테스트 자동화)은 WBS 2부터 단계별 병행 가능
3. WBS 9(배포/운영 검증)은 WBS 8 완료 후 진행

## 7. 리스크/결정 백로그

1. 그룹 정렬(position) 배치 변경 정책 확정
2. 오프라인 편집/재연결 정책 확정
3. 대량 정렬/장문 편집 성능 상한(지연 허용치, 타임아웃) 확정
