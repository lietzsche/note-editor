## 4. 타임스탬프 메모

현재 코드/DDL의 시간 저장 기준은 KST가 아니라 UTC다.

근거:
- 애플리케이션에서 직접 넣는 시간은 `new Date().toISOString()`이며 UTC ISO-8601 형식이다.
- SQLite `datetime('now')`의 `now`는 UTC 기준이다.
- Cloudflare D1는 SQLite 의미론을 따른다.

해석:
- Cloudflare에 적용되어도 DB 기본 시간은 한국시간으로 자동 저장되지 않는다.
- 따라서 대시보드나 SQL 조회 결과를 보면 KST보다 9시간 차이 나 보일 수 있다.
- 이 동작은 현재 구현 버그라기보다 저장 기준이 UTC이기 때문에 발생하는 정상 동작이다.

권장 정책:
- 저장은 UTC로 통일
- 화면 표시와 운영 리포트에서만 KST로 변환

주의:
- 현재 스키마 기본값 `datetime('now')`는 `YYYY-MM-DD HH:MM:SS` 형식이고,
  앱에서 직접 넣는 값은 `YYYY-MM-DDTHH:MM:SS.sssZ` 형식이다.
- 둘 다 UTC이지만 문자열 형식이 달라 운영 조회 시 혼동될 수 있다.
- 추후 한 포맷으로 통일하는 정리가 필요하다.
- 한국시간 저장 전환 여부는 후속 검토 항목으로 보류한다.

## 4.1 라우팅 메모

- SPA 새로고침 라우팅은 `_redirects`가 아니라 `wrangler.toml`의 Workers assets 설정으로 처리한다.
- 현재 기준 설정은 `assets.binding = "ASSETS"` + `not_found_handling = "single-page-application"`이다.
- `_redirects`로 `/* /index.html 200` 규칙을 추가하면 Cloudflare 배포 검증에서 infinite loop 오류가 발생할 수 있다.
