- 정렬 payload 전체 노트 ID 배열 검증

## 3. DB / 마이그레이션 메모

이번 WBS 3에서는 새 마이그레이션을 추가하지 않았다.

근거:
- `pages` 테이블에 `title`, `content`, `group_id`, `sort_order`, `updated_at`가 이미 존재한다.
- 이번 범위의 핵심 변경은 스키마 추가보다 API 검증과 `sort_order` 갱신 로직이다.

주의:
- 길이 제한과 payload 검증은 현재 API 레이어에서 강제한다.
- DB 레벨 `CHECK` 제약까지 강제할지는 후속 마이그레이션 검토 항목으로 둔다.

## 4. 디자인 검토 메모

WBS 7 병행 점검 기준으로 다음 항목을 함께 보정했다.

- 토큰 별칭(`--color-*`, `--radius`)을 유지해 기존 화면과 시각 일관성 유지
- 그룹/노트 선택을 키보드 접근 가능한 `button` 구조로 교체
- 아이콘성 버튼에 `aria-label` 추가
- 모바일 폭(`max-width: 900px`)에서 `그룹 / 노트 / 편집` 단일 패널 전환 지원
- 저장/정렬/복사 상태를 텍스트로 노출해 색상 외 채널로도 상태 전달

남은 메모:
- 현재 Notes 화면은 단일 페이지 내 인라인 스타일 중심 구현이다.
- 추후 공용 컴포넌트(`GroupList`, `NoteList`, `NoteEditor`) 분리 필요 여부를 WBS 4~6 진행 중 재검토한다.
