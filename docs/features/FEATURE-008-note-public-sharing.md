# FEATURE-008: 노트 공개 공유(Note Public Sharing)

- 문서 버전: v1.0
- 작성일: 2026-04-21
- 연계 PRD: (추가 예정)

## 1. 목적/범위

사용자가 특정 노트를 공개적으로 공유할 수 있도록 하여, 로그인하지 않은 사용자도 URL을 통해 읽기 전용으로 노트 내용을 볼 수 있게 한다. 공유된 노트는 수정할 수 없으며, 공유를 해제하면 해당 URL 접근이 차단된다.

## 2. 사용자 시나리오

1. 사용자가 특정 노트에 대한 공개 공유를 활성화한다.
2. 시스템이 고유한 공유 URL을 생성하여 제공한다.
3. 사용자가 해당 URL을 다른 사람에게 전달한다.
4. 로그인하지 않은 사용자가 공유 URL을 통해 노트 내용을 읽기 전용으로 확인한다.
5. 사용자가 필요시 공유를 비활성화하여 더 이상 접근을 허용하지 않는다.

## 3. 기능 요구사항

1. `공유 활성화`: 사용자가 특정 노트에 대한 공개 공유를 켜거나 끌 수 있다.
2. `공유 URL 생성`: 노트 공유가 활성화되면 고유한 식별자를 포함한 URL을 생성한다 (예: `/shared/{noteId}/{shareToken}`).
3. `읽기 전용 접근`: 공유된 노트는 로그인 여부와 관계없이 읽기만 가능하며, 수정, 삭제, 공유 설정 변경은 불가능하다.
4. `공유 상태 관리`: 노트 소유자는 언제든지 공유를 비활성화할 수 있으며, 비활성화 시 즉시 접근이 차단된다.
5. `접근 제어`: 공유된 노트는 인증 없이 접근 가능하지만, 노트 소유자만 공유 설정을 변경할 수 있다.
6. `공유 정보 표시`: 노트 편집 화면에서 현재 공유 상태와 공유 URL을 확인할 수 있다.
7. `보안 고려사항`: 공유 토큰은 예측 불가능한 충분한 엔트로피를 가져야 하며, 노트 ID만으로 접근이 가능하지 않도록 한다.
8. `만료 옵션`: 필요시 공유 기한을 설정할 수 있는 옵션을 제공한다 (기본값: 무제한).
9. `접근 통계`: 공유된 노트에 대한 접근 횟수나 시간을 기록할 수 있는 옵션을 제공한다 (선택 사항).
10. `편집 화면 배치`: 공유 제어는 편집 생산성 액션과 같은 보조 액션 영역에 배치하며, 모바일과 데스크톱 모두 본문 편집 공간을 밀어내는 상세 패널을 기본 노출하지 않는다.

## 4. API/데이터 계약

1. 엔드포인트
- `POST /api/notes/{noteId}/share` - 노트 공유 활성화/갱신
- `DELETE /api/notes/{noteId}/share` - 노트 공유 비활성화
- `GET /api/notes/{noteId}/share` - 노트 공유 상태 조회
- `GET /api/shared/{shareToken}` - 공유된 노트 읽기 (인증 불필요)

2. 데이터 구조
```json
{
  "noteId": "string",
  "shareToken": "string",
  "isActive": "boolean",
  "createdAt": "timestamp",
  "expiresAt": "timestamp | null",
  "accessCount": "number"
}
```

3. 응답 원칙
- 성공: 공유 정보 또는 노트 내용 반환
- 실패: `{"error":{"code":"...","message":"..."}}`

## 5. 상태/예외 처리

1. 존재하지 않는 노트 -> 404 Not Found
2. 권한 없음 (소유자가 아닌 사용자가 공유 설정 변경 시도) -> 403 Forbidden
3. 공유 토큰이 유효하지 않거나 만료됨 -> 404 Not Found 또는 410 Gone
4. 공유가 비활성화된 노트 접근 -> 404 Not Found
5. 공유된 노트에 대한 수정 시도 -> 405 Method Not Allowed

## 6. 테스트 기준(TDD 포인트)

1. Red
- 소유자가 아닌 사용자가 공유 설정 변경 시도 시 실패
- 존재하지 않는 노트 공유 시도 시 실패
- 비활성화된 공유 링크 접근 시 실패

2. Green
- 노트 소유자가 공유 활성화/비활성화 성공
- 공유된 노트 인증 없이 읽기 성공
- 공유 상태 정상 조회

3. Refactor
- 공유 토큰 생성 로직 분리 및 보안 강화
- 접근 제어 미들웨어 리팩토링

## 7. 오픈 이슈

1. 공유 링크의 접근 제어 수준 (비밀번호 보호 옵션 필요 여부)
2. 공유 통계 수집 범위 결정
3. 공유 만료 기본값 설정
4. 공유된 노트에 대한 검색 엔진 색인 방지 방법 (noindex 메타 태그 등)

## 8. 2026-04-30 UI 정리 메모

1. 공유 상태 조작은 축약 버튼으로 제공한다.
2. 공유가 켜진 경우 공유 링크 열기 액션을 함께 노출한다.
3. 공유 처리 오류는 축약 영역 안에서 텍스트로 노출해 색상만으로 전달하지 않는다.
