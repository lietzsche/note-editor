# FEATURE-005: 편집 생산성(Editor Productivity)

- 문서 버전: v1.0
- 작성일: 2026-04-11
- 연계 PRD: SR-10, SR-11

## 1. 목적/범위

노트 편집 중 가시성과 작업 속도를 높이기 위해 실시간 글자 수 표시와 노트 전체 복사 기능을 제공한다.

## 2. 사용자 시나리오

1. 사용자가 본문을 입력/삭제하며 실시간 글자 수를 확인한다.
2. 사용자가 노트 A/B를 전환하며 각 노트의 글자 수를 확인한다.
3. 사용자가 복사 버튼으로 현재 노트 전체 본문을 클립보드로 복사한다.
4. 복사 성공/실패 피드백을 확인하고 후속 작업을 진행한다.

## 3. 기능 요구사항

1. 실시간 글자 수는 본문 변경 즉시 갱신되어야 한다.
2. 글자 수는 현재 선택된 노트 본문 기준으로 표시되어야 한다.
3. 전체 복사는 명시적 버튼 액션으로만 실행되어야 한다.
4. 복사 성공/실패 피드백은 UI에 명확히 노출되어야 한다.
5. 빈 본문 처리 정책을 명확히 유지해야 한다.
- 기본 정책: 빈 본문도 복사 액션은 허용하고, 성공 피드백을 표시한다.
6. 글자 수 계산은 공백/줄바꿈을 포함한 grapheme cluster 기준으로 처리한다.
7. 편집 툴바는 제목/그룹 선택 영역과 저장 상태/생산성 액션 영역을 분리한다.
8. 모바일에서는 툴바 정보가 본문 편집 공간을 과도하게 줄이지 않도록 보조 액션을 아이콘 중심으로 압축한다.

## 4. API/데이터 계약

1. 서버 API 추가는 필수가 아니다.
2. 글자 수 계산은 클라이언트 상태를 기준으로 처리한다.
3. 전체 복사는 브라우저 Clipboard API를 사용한다.
4. 구현 단위
- `countGraphemes(text)`: grapheme cluster 계산 유틸
- `copyText(clipboard, text)`: Clipboard API 래퍼
- `CharacterCountIndicator`, `CopyAllButton`: 보조 UI 컴포넌트

## 5. 상태/예외 처리

1. 글자 수 표시 상태
- `count-ready`: 정상 표시
- `count-stale`: 노트 전환 직후 갱신 대기(짧은 과도 상태 허용)

2. 복사 상태
- `copy-success`: 복사 성공 피드백
- `copy-error`: 복사 실패 피드백 + 재시도 유도

3. 실패 케이스
- 브라우저 권한 제한/보안 컨텍스트 제한 시 실패 피드백

## 6. 테스트 기준(TDD 포인트)

1. Red
- 입력/삭제 시 글자 수 미갱신 실패 테스트 선작성
- 노트 전환 시 글자 수 불일치 실패 테스트 선작성
- 복사 성공/실패 분기 테스트 선작성

2. Green
- 글자 수 실시간 표시 및 복사 버튼 기능 최소 구현 통과

3. Refactor
- 글자 수 계산 유틸과 클립보드 핸들러를 분리

## 9. 2026-04-30 UI 정리 메모

1. 저장 상태와 글자 수는 같은 상태 클러스터에 배치해 현재 노트의 편집 상태를 한 번에 확인하게 한다.
2. 전체 복사, 맞춤법, 검색 정보, 공유 액션은 보조 액션 클러스터에 배치한다.
3. 공유 상세 정보는 기본 편집 흐름을 밀어내지 않도록 축약 액션으로 제공하며, 활성 링크 열기와 공유 켜기/끄기 액션의 접근 가능한 이름을 유지한다.
