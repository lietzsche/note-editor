# FEATURE-004: 노트 전환 독립성/무결성

- 문서 버전: v1.0
- 작성일: 2026-04-11
- 연계 PRD: SR-9

## 1. 목적/범위

노트 전환 과정에서 내용 혼선과 유실을 방지하고, 저장 실패/충돌 상황을 안전하게 처리한다.

## 2. 사용자 시나리오

1. 사용자가 노트 A/B를 반복 전환한다.
2. 미저장 상태에서 다른 노트로 이동한다.
3. 저장 실패 후 재시도를 수행한다.
4. 다중 탭 편집 충돌을 처리한다.

## 3. 기능 요구사항

1. 노트 A/B 전환 시 내용이 서로 섞이지 않아야 한다.
2. 미저장 변경 상태(`dirty`)를 추적해야 한다.
3. 미저장 상태 전환 시 `저장 후 이동 / 버리고 이동 / 취소`를 제공해야 한다.
4. 저장 실패 시 마지막 입력값을 유지하고 재시도 가능해야 한다.
5. 충돌 감지 시 `conflict` 상태로 전환하고 `덮어쓰기/취소` 선택 UI를 제공해야 한다.
6. 취소 선택 시 로컬 수정 내용은 유지되어야 한다.

## 4. API/데이터 계약

1. 노트 단건 업데이트는 `updated_at` 기반 낙관적 충돌 감지를 사용한다.
2. 요청/응답 필드(핵심)
- `PATCH /api/notes/:id`
- request body: `title`, `content`, `group_id?`, `updated_at?`, `force?`
- `updated_at`: 마지막으로 동기화한 서버 수정 시각
- `force: true`: 충돌 확인 후 로컬 수정본으로 강제 저장

3. 충돌 응답 규격
- `409 CONFLICT`
- response body: `error.code = "CONFLICT"` + `data`에 서버 최신 노트 스냅샷 포함

4. 에러 코드(권장)
- `CONFLICT`
- `VALIDATION_ERROR`
- `UNAUTHORIZED`

## 5. 상태/예외 처리

1. `dirty` + 전환 시도 -> 전환 확인 다이얼로그
2. 저장 실패 -> `error` + 재시도 CTA
3. 충돌 -> `conflict` 상태에서 선택 완료 전 자동 덮어쓰기 금지
4. 명시적 삭제 액션 없이 노트 삭제 금지
