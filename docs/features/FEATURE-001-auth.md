# FEATURE-001: 인증(Auth)

- 문서 버전: v1.0
- 작성일: 2026-04-11
- 연계 PRD: SR-1, SR-7

## 1. 목적/범위

회원가입, 로그인, 로그아웃, 세션 확인 기능을 제공하고 보호 API 접근 제어를 보장한다.

## 2. 사용자 시나리오

1. 신규 사용자가 가입 후 로그인한다.
2. 로그인 사용자가 보호 API를 호출한다.
3. 로그아웃 후 보호 API 접근이 차단된다.
4. 세션 만료 후 재로그인으로 접근을 복구한다.

## 3. 기능 요구사항

1. `signup`은 유효한 입력으로 계정을 생성한다.
2. `login`은 성공 시 세션을 발급한다.
3. `logout`은 현재 세션을 무효화한다.
4. `me`는 현재 로그인 사용자 정보를 반환한다.
5. 인증 실패는 일관된 에러 포맷으로 반환한다.
6. 세션 만료 정책은 기본 `고정 만료(absolute TTL, 7일)`로 적용한다.
7. 자동 갱신(sliding TTL)은 기본 비활성화하며 필요 시 명시적으로 활성화한다.
8. 세션 쿠키는 `HttpOnly`, `SameSite=Lax`를 기본으로 하고 운영 환경에서 `Secure`를 강제한다.
9. 로그인 실패 시 계정/IP 기준 속도 제한(기본: 분당 5회, 초과 시 5분 제한)을 적용한다.
10. 인증 이벤트(성공/실패/로그아웃)를 감사 로그로 기록한다.

## 4. API/데이터 계약

1. 엔드포인트
- `POST /api/auth/signup`
- `POST /api/auth/login`
- `POST /api/auth/logout`
- `GET /api/auth/me`

2. 응답 원칙
- 성공: 사용자/세션 상태 반환
- 실패: `{"error":{"code":"...","message":"..."}}`

## 5. 상태/예외 처리

1. 잘못된 자격증명 -> 로그인 실패
2. 인증 없음 -> 보호 API 차단
3. 세션 만료 -> 재로그인 유도
