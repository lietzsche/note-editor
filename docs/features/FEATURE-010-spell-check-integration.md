# FEATURE-010: 맞춤법 검사 통합(Spell Check Integration)

- 문서 버전: v1.0
- 작성일: 2026-04-21
- 연계 PRD: (추가 예정)

## 1. 목적/범위

사용자가 노트 편집 중에 맞춤법 검사를 쉽게 수행할 수 있도록 외부 맞춤법 검사 사이트(https://nara-speller.co.kr/speller/)로 연결하는 버튼 기능을 제공한다. 이 기능은 노트 내용을 외부 사이트에 전송하여 검사를 받을 수 있도록 하는 간편한 통합을 목표로 한다.

## 2. 사용자 시나리오

1. 사용자가 노트 편집 중 맞춤법 검사가 필요하다고 생각한다.
2. 편집기 툴바나 메뉴에서 "맞춤법 검사" 버튼을 클릭한다.
3. 시스템이 현재 노트 내용을 외부 맞춤법 검사 사이트로 전송하거나, 사이트를 새 탭/창으로 열어 사용자가 직접 내용을 붙여넣을 수 있도록 안내한다.
4. 사용자는 외부 사이트에서 맞춤법 검사를 수행하고 수정된 내용을 노트 편집기에 다시 복사한다.

## 3. 기능 요구사항

1. `맞춤법 검사 버튼`: 노트 편집기 UI에 맞춤법 검사를 위한 버튼 또는 메뉴 항목을 추가한다.
2. `외부 사이트 연결`: 버튼 클릭 시 https://nara-speller.co.kr/speller/ 사이트로 연결한다.
3. `내용 자동 전송 옵션`: 가능하다면 현재 노트 내용을 외부 사이트의 입력란에 자동으로 채워넣을 수 있는 기능을 제공한다 (사이트 API 지원 여부에 따라).
4. `새 탭/창 열기`: 외부 사이트는 새 브라우저 탭이나 창에서 열리도록 하여, 사용자가 노트 편집 화면과 검사 결과를 동시에 볼 수 있게 한다.
5. `사용자 안내`: 버튼 클릭 시 외부 사이트 사용 방법에 대한 간단한 안내를 제공한다 (예: "검사 후 수정된 내용을 복사하여 편집기에 붙여넣어주세요").
6. `접근성`: 버튼은 키보드 접근 가능하고, 스크린 리더 사용자를 위한 적절한 라벨과 설명을 제공한다.
7. `UI 통합`: 버튼 디자인은 기존 편집기 UI와 조화를 이루도록 한다.

## 4. 구현 방식

1. **간단한 구현 (1단계)**
   - 하이퍼링크 버튼: `<a href="https://nara-speller.co.kr/speller/" target="_blank" rel="noopener noreferrer">맞춤법 검사</a>`
   - 버튼 클릭 시 새 탭에서 맞춤법 검사 사이트 열기
   - 사용자가 수동으로 내용을 복사/붙여넣기

2. **고도화 구현 (향후)**
   - 현재 편집기 내용을 자동으로 외부 사이트로 전송
   - 외부 사이트 API 연동 (가능한 경우)
   - 검사 결과를 편집기에 자동으로 반영하는 기능

## 5. UI/UX 요구사항

1. 버튼 위치: 편집기 툴바, 상단 메뉴, 또는 부가 기능 영역
2. 버튼 텍스트: "맞춤법 검사" 또는 "맞춤법 검사기"
3. 아이콘: 맞춤법 검사를 상징하는 적절한 아이콘 (선택 사항)
4. 툴팁: "외부 맞춤법 검사 사이트로 이동합니다"

## 6. 보안 고려사항

1. `rel="noopener noreferrer"`: 새 탭에서 열릴 때 보안을 위해 반드시 포함
2. `사용자 데이터 보호`: 외부 사이트로 내용을 전송할 경우 사용자에게 명시적 동의 요청
3. `HTTPS`: 외부 사이트 연결 시 HTTPS 프로토콜 사용 확인

## 7. 테스트 기준(TDD 포인트)

1. Red
- 버튼 클릭 시 올바른 URL로 연결되는지 확인
- 새 탭에서 열리는지 확인
- 보안 속성(noopener, noreferrer)이 적용되었는지 확인

2. Green
- 버튼 UI 구현 및 기능 동작 확인
- 다양한 브라우저에서 정상 작동 확인

3. Refactor
- 버튼 컴포넌트 추상화
- 외부 링크 관리를 위한 설정 파일 분리

## 8. 오픈 이슈

4. 오프라인 맞춤법 검사 기능 필요성 검토 (장기적)
