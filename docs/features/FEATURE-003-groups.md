# FEATURE-003: 그룹(Group) 기반 관리

- 문서 버전: v1.0
- 작성일: 2026-04-11
- 연계 PRD: SR-8

## 1. 목적/범위

노트를 그룹 단위로 분류하고 탐색하는 기능을 제공한다.

## 2. 사용자 시나리오

1. 사용자가 그룹을 생성한다.
2. 그룹별로 노트를 생성/조회한다.
3. 노트를 다른 그룹으로 이동한다.
4. 그룹 이름을 변경한다.
5. 그룹을 삭제하고 소속 노트 처리 정책을 확인한다.

## 3. 기능 요구사항

1. 그룹 생성/조회/수정/삭제가 가능해야 한다.
2. 그룹 필터 조회 시 해당 그룹 노트만 반환해야 한다.
3. 그룹 미지정 상태(기본 그룹)도 정상 지원해야 한다.
4. 그룹 삭제 시 소속 노트는 기본 그룹(미분류)으로 이동한다.
5. 기본 그룹은 삭제할 수 없다.

## 4. API/데이터 계약

1. 엔드포인트(권장)
- `GET /api/groups`
- `POST /api/groups`
- `PUT /api/groups/:id`
- `DELETE /api/groups/:id`
- `PATCH /api/pages/:id/group`

2. 데이터 필드(핵심)
- `groups`: `id`, `user_id`, `name`, `position`, `created_at`, `updated_at`
- `notes/pages`: `group_id`

## 5. 상태/예외 처리

1. 그룹명 중복 -> 같은 사용자 내 중복 금지
2. 존재하지 않는 그룹 -> 이동/수정/삭제 실패
3. 타 사용자 그룹 접근 -> 권한 오류
4. 삭제 대상 그룹이 기본 그룹일 때 -> 삭제 제한 또는 정책 명시 필요

## 6. 테스트 기준(TDD 포인트)

1. Red
- 중복 그룹명, 잘못된 group_id, 타 사용자 접근 실패 테스트 선작성

2. Green
- 그룹 CRUD 및 그룹 필터 조회 최소 구현 통과

3. Refactor
- 그룹 권한 검증과 노트 이동 정책 로직 분리

## 7. 오픈 이슈

1. 기본 그룹의 생성 시점(가입 시 자동 생성 vs 최초 그룹 액션 시 생성) 확정 필요
2. 그룹 최대 개수/이름 길이 제한 확정 필요
