# Note Editor Meta Fix Bug

## 목적

버그를 성급히 덮지 않고, 재현과 영향 범위를 확인한 뒤 수정한다. 수정 후에는 코드 리뷰 관점으로 회귀를 다시 점검하고, 동작이나 운영 지식이 바뀌면 문서까지 맞춘다.

## 체인 순서

1. 오류를 재현하거나 로그, 테스트, 코드 경로를 따라가며 원인과 영향 범위를 확인한다.
2. `$note-editor-apply-feedback`으로 수정 코드와 회귀 테스트를 반영한다.
3. `$note-editor-code-review`로 수정이 다른 흐름을 깨지 않는지, 놓친 분기가 없는지 점검한다.
4. 다시 `$note-editor-apply-feedback`으로 리뷰 결과와 문서 갱신을 반영한다.

## 실행 규칙

1. 재현이 가능하면 재현 절차와 기대 동작, 실제 동작을 먼저 적는다.
2. 재현이 불완전해도 가장 강한 가설과 확인 방법을 남긴다.
3. 수정은 증상 완화보다 원인 제거를 우선한다.
4. 버그가 계정 격리, 데이터 유실, 자동 저장, 캐시 동기화에 닿으면 관련 경로를 넓게 검토한다.
5. 문서화는 사용자-facing 동작이 바뀌었을 때만 최소 범위로 남긴다. 내부 운영 지식이 바뀌면 운영 문서도 갱신한다.

## 기본 산출물

- 재현 조건 또는 원인 가설.
- 수정 코드와 회귀 테스트.
- 코드 리뷰 finding 및 후속 반영 결과.
- 갱신된 기능/운영/테스트 문서.

## 예시 요청

- “`$note-editor-meta-fix-bug`를 사용해서 노트 저장 중 다른 노트로 전환하면 내용이 섞이는 오류를 확인하고 수정해줘.”
- “`$note-editor-meta-fix-bug`로 공유 링크 만료일이 저장되지 않는 버그를 재현부터 문서화까지 끝내줘.”
