# Note Editor Meta Add Feature

## 목적

이미 운영 중인 프로젝트에 새 기능을 붙일 때 범위 누락과 회귀를 줄인다. 기존 구조를 읽고 기능을 설계한 뒤, 구현과 검토, 문서 동기화까지 한 번에 마무리한다.

## 체인 순서

1. `$note-editor-product-planning`으로 기존 기능과 충돌하지 않는 요구사항, 비범위, 수용 기준을 정리한다.
2. `$note-editor-apply-feedback`으로 기능 추가 구현과 테스트 초안을 반영한다.
3. `$note-editor-design-review`로 새 UI가 기존 UX와 디자인 시스템에 맞는지 점검한다.
4. `$note-editor-code-review`로 회귀 위험, 데이터 무결성, 계정 격리, 테스트 누락을 점검한다.
5. 다시 `$note-editor-apply-feedback`으로 리뷰 결과와 문서 갱신을 반영한다.

## 실행 규칙

1. 기존 동작을 깨지 않는 호환성 관점을 먼저 본다.
2. feature 문서가 이미 있으면 새 요구사항을 거기에 합치고, 없으면 필요한 문서를 만든다.
3. 추가 기능이 API, DB, 자동 저장, reorder, 공유 링크에 닿으면 관련 회귀 테스트를 우선 추가한다.
4. 디자인 리뷰는 “새 기능이 잘 보이는가”보다 “기존 편집 흐름을 방해하지 않는가”를 우선 본다.
5. 문서화는 기능 설명뿐 아니라 새 수용 기준과 검증 시나리오를 남긴다.

## 기본 산출물

- 기존 프로젝트 맥락을 반영한 기능 요구사항.
- 추가 기능 구현과 회귀 테스트.
- 디자인/코드 리뷰 finding 및 반영 결과.
- 갱신된 feature 문서와 테스트 계획.

## 예시 요청

- “`$note-editor-meta-add-feature`를 사용해서 기존 노트 상세 화면에 공유 링크 만료일 기능을 추가해줘.”
- “`$note-editor-meta-add-feature`로 태그 필터 기능을 기획부터 구현, 리뷰, 문서화까지 진행해줘.”
