# WBS: note-editor-v2 개발 작업 분해 구조

- 문서 버전: v2.0
- 작성일: 2026-04-21
- 기준 문서: `docs-v2/product/PRD.md`, `docs-v2/testing/TEST_PLAN.md`, `docs-v2/specs/*`, `docs-v2/operations/DEPLOY.md`

## 1. 목적

SR-1 ~ SR-11을 구현 가능한 단위로 분해하고, 선행관계/산출물/완료기준을 명확히 해 개발 착수와 진행 관리를 표준화한다.

## 2. 범위

1. V1 핵심 기능 (SR-1 ~ SR-11): 인증, 노트 CRUD, 그룹 관리, 전환 독립성, 편집 생산성
2. V2 확장 기능 (SR-12 ~ SR-14): PWA 지원, 검색 및 태그 시스템, 노트 공유 기능
3. 테스트 자동화 체계 구축

## 3. 단계별 WBS

| WBS ID | 작업 | 상세 작업 | 산출물 | 선행 | 완료 기준 |
|---|---|---|---|---|---|

## 6. 병행 가능 작업

1. WBS 7(단계별 디자인 검토/보정)은 독립 후행 단계가 아니라 WBS 2~6 각 단계 종료 시 반복 수행
2. 각 기능 WBS 완료 전, 해당 화면의 토큰/컴포넌트/접근성/반응형 최소 점검을 함께 수행
3. WBS 8(테스트 자동화)은 WBS 2부터 단계별 병행 가능

단계 종료 시 디자인 검토 항목:
1. 새로 추가되거나 수정된 화면이 토큰 체계를 사용하고 있는지 확인
2. 공용 컴포넌트로 흡수할 수 있는 스타일이 생겼는지 확인
3. 키보드 포커스, 상태 표시, 오류 메시지, 모바일 폭에서의 사용성을 최소 점검
4. 디자인 문서/체크리스트에 반영이 필요한 변경이 있으면 같은 턴에 함께 갱신

반영 로그:
- WBS 2 디자인 검토/보정 기록: `docs/STATUS-2026-04-11.md`
- WBS 3~4 디자인 검토/보정 기록: `docs/STATUS-2026-04-12.md`

## 7. 리스크/결정 백로그

1. 그룹 정렬(position) 배치 변경 정책 확정
2. 오프라인 편집/재연결 정책 확정
3. 대량 정렬/장문 편집 성능 상한(지연 허용치, 타임아웃) 확정
4. 모바일 설치형 앱(PWA) 지원 범위 확정: 웹 앱 매니페스트, 설치용 아이콘, 홈 화면 설치 진입, iOS/Android 동작 차이 정리
5. PWA 설치 지원은 현재 릴리스 범위 밖 후속 백로그로 두고, 범위 확정 후 구현 태스크로 분해
