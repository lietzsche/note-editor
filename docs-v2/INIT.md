# V2 프로젝트 초기화 문서

## 개요

이 문서는 노트 편집기 V2 프로젝트의 초기 설정 및 시작점을 기록합니다. V2는 V1의 핵심 기능을 유지하면서 관리자 기능, 노트 공유, PWA 지원 등 새로운 기능을 추가하며, Cloudflare Workers와 D1 데이터베이스를 기반으로 재구축됩니다.

## 프로젝트 배경

### V1에서의 교훈
- **장점**: React + TypeScript 기반의 모던한 프론트엔드, DnD 기반의 직관적인 UI
- **아쉬운 점**: 인증 및 권한 관리 부재, 관리자 기능 미흡, 오프라인 지원 부족
- **기술적 한계**: 단일 페이지 애플리케이션 구조, 확장성 제한

### V2 목표
1. **확장성**: 다중 사용자 지원 및 권한 관리 체계 구축
2. **관리성**: 관리자 대시보드 및 시스템 모니터링 기능 추가
3. **접근성**: 공개 노트 공유 및 PWA 지원으로 다양한 접근 방식 제공
4. **유지보수성**: 모듈화된 아키텍처 및 체계적인 문서화

## 기술적 결정

### 프론트엔드
- **프레임워크**: React 18 (함수형 컴포넌트, Hooks)
- **상태 관리**: Context API + useReducer (복잡도에 따라 Zustand 도입 검토)
- **스타일링**: CSS Modules + 디자인 토큰 시스템
- **라우팅**: React Router v6
- **빌드 도구**: Vite
- **테스트**: Vitest + React Testing Library

### 백엔드
- **프레임워크**: Hono (라이트웨이트 웹 프레임워크)
- **데이터베이스**: Cloudflare D1 (SQLite)
- **인증**: JWT + 세션 관리
- **API 디자인**: RESTful + OpenAPI 명세
- **테스트**: Vitest + Cloudflare Workers 테스트 환경

### 인프라
- **호스팅**: Cloudflare Workers (서버리스)
- **데이터베이스**: Cloudflare D1
- **스토리지**: Cloudflare R2 (미디어 파일)
- **캐싱**: Cloudflare KV
- **모니터링**: Cloudflare Analytics
- **도메인**: Cloudflare DNS 관리
